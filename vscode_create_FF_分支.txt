前置条件
已完成上一步的操作（本地仓库已关联 GitHub 远程仓库，且 main 分支能正常推送）
VS Code 终端已打开（快捷键 `Ctrl +``）
当前处于 main 分支（可通过 git branch 确认，带 * 的是当前分支）
完整操作实例（指令 + 详解）
步骤 1：确保 main 分支是最新的（关键！避免合并冲突）
在创建新分支前，先拉取远程 main 分支的最新代码，保证本地 main 分支和远程一致：
git pull origin main
步骤 2：创建并切换到 FF 分支
bash
运行
git checkout -b FF
指令含义：
git checkout 原本是 “切换分支” 的指令；
-b 是 "branch" 的缩写，作用是创建新分支并立即切换到该分支（等价于先执行 git branch FF 创建分支，再执行 git checkout FF 切换分支）；
FF 是你要创建的分支名称（分支名建议见名知意，比如 feature-login、fix-bug 等）
步骤 3：在 FF 分支编写 / 修改代码
这一步无需指令，直接在 VS Code 中编写你的代码（比如新增文件 test.js、修改 index.html 等）。
步骤 4：将 FF 分支的修改提交到本地仓库
bash
运行
# 1. 查看修改状态（可选，确认要提交的文件）
git status

# 2. 将所有修改添加到暂存区
git add .

# 3. 提交到本地仓库
git commit -m "FF分支：新增test.js文件，优化index.html样式"
指令含义：
git status：查看工作区哪些文件被修改 / 新增，确认无误后再提交；
git add .：将当前分支下所有修改 / 新增的文件加入暂存区；
git commit -m "说明"：将暂存区的内容提交到本地 FF 分支，提交说明要明确标注分支和修改内容，方便后续追溯。
步骤 5：将 FF 分支推送到 GitHub 远程仓库
bash
运行
git push -u origin FF
指令含义：
git push：推送本地分支到远程；
-u：设置默认上游分支（后续在 FF 分支推送只需执行 git push）；
origin：远程仓库别名；
FF：要推送的本地分支名（远程会自动创建同名的 FF 分支）。
步骤 6：切换回 main 分支
合并分支前必须先切换到目标分支（main）：
bash
运行
git checkout main//合并FF分支之前，要切回main分支
指令含义：git checkout main 切换到本地 main 分支，确保后续合并操作是基于 main 分支执行。
步骤 7：将 FF 分支合并到 main 分支（本地）
bash
运行
git merge FF
指令含义：
git merge 是 “合并分支” 的核心指令；
FF 是要合并到当前分支（main）的源分支名；
执行后，Git 会将 FF 分支的所有提交记录合并到本地 main 分支。
✨ 注意：如果合并时提示冲突（比如 FF 和 main 都修改了同一个文件的同一行），需先在 VS Code 中解决冲突（VS Code 会高亮冲突区域，选择保留哪部分代码），解决后执行 git add . 和 git commit -m "解决合并冲突，合并FF分支到main" 完成合并。
步骤 8：将合并后的 main 分支推送到 GitHub
bash
运行
git push
//这里执行完这个语句之后，是需要去github里去同意FF分支到main的合并的，
//不然main里没有FF分支的内容
指令含义：因为之前 main 分支已设置过上游（-u origin main），直接执行 git push 即可将合并后的本地 main 分支推送到远程 GitHub 的 main 分支，完成最终合并。
（可选）删除本地 / 远程 FF 分支（分支合并后无需保留时）
bash
运行
# 删除本地 FF 分支
git branch -d FF

# 删除远程 GitHub 上的 FF 分支
git push origin --delete FF
指令含义：
git branch -d FF：-d 是 "delete" 的缩写，删除本地已合并的分支（如果分支未合并，需用 -D 强制删除）；
git push origin --delete FF：删除远程仓库中的 FF 分支。
