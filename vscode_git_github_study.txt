VS Code 推送本地代码到 GitHub：完整指令与步骤


0) 参考顺序（最常用快速步骤）

```bash
# 在本地新项目中快速执行：
git init
git add .   //把文件放到暂存区，用于提交
git commit -m "Initial commit"   //提交暂存区里的文件，提交的名称为"Initial commit"
git branch -M main
git remote add origin <REPO_URL> //<REPO_URL>是git仓库的SSH
git push -u origin main  //push 推送的意思，也就是把电脑本地的文件推送到github上的仓库
```
1) 全局配置（只需执行一次）

```bash
# 设置用户名与邮箱（替换为你的信息）
git config --global user.name "你的姓名"//**qwwmnn123**
git config --global user.email "you@example.com"//**164953341@qq.com**



2) 在本地项目目录初始化仓库


git init
# 创建 .gitignore（如果需要）
//.gitignore里面放的文件名，都是不需要上传和下载的文件


3) 首次提交（添加、提交）

```bash
git add .  //**本地文件提交到暂存去的所有文件把暂存区的**
git commit -m "Initial commit"//**"Initial commit"只是个名字，可以为"第一次提交等"**
```

4) 在 GitHub 创建远程仓库（在浏览器里），然后关联远程并推送

- 在 GitHub 网站上创建仓库（不勾选初始化 README，否则会出现合并冲突）。
- 假设远程仓库 URL 为：https://github.com/USERNAME/REPO.git 或 git@github.com:USERNAME/REPO.git

```bash
# 设置远程 origin（替换为你的仓库 URL）
git remote add origin <REPO_URL>
# 将本地 main 分支推到远程并建立上游分支
git branch -M main
git push -u origin main
```

9) 常见的推拉命令（日常使用）

```bash
# 查看状态
git status
# 拉取远端最新并合并到当前分支
git pull
# 或使用 rebase：
git pull --rebase //把远程github上的仓库内容拉倒本地，保持本地仓库的文件和github上最新的保持一致，防止下次推送的时候报错
